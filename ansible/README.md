## 🎯 How to Use These Projects

### 1️⃣ Choose a Project
- Browse by **category** or **difficulty level**
- Each project has a dedicated folder: `project_001/` through `project_100/`

### 2️⃣ Navigate to Project Directory
```bash
cd ansible/project_001
```

### 3️⃣ Read the Project README
```bash
cat README.md
```

Each project README contains:
- **Overview**: What the project does
- **Business Use-Case**: Real-world scenario
- **Prerequisites**: Required tools and knowledge
- **Implementation Steps**: Detailed instructions
- **Expected Output**: What success looks like
- **Challenges**: Learning objectives
- **Extensions**: Ideas to expand the project

### 4️⃣ Install Dependencies
```bash
ansible-galaxy install -r requirements.txt  # if roles are listed
pip install -r requirements.txt             # if Python modules needed
```

### 5️⃣ Implement & Learn
Follow the project steps, customize for your environment, and experiment!

## 📝 Project Structure Template

Each project follows this structure:
```
project_XXX/
├── README.md           # Project documentation
├── requirements.txt    # Dependencies (roles, collections, packages)
├── src/                # Playbooks, roles, and scripts
│   ├── playbook.yml
│   ├── inventory/
│   ├── roles/
│   └── group_vars/
└── assets/             # Supporting files (configs, templates, data)
```
