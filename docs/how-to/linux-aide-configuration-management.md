# Linux AIDE Configuration Management

## Purpose

Deploy and manage AIDE (Advanced Intrusion Detection Environment) for file integrity monitoring on Linux systems. This guide covers automated installation, configuration, baseline creation, and continuous monitoring with scheduled integrity checks.

## When to use

- Establishing file integrity monitoring on production Linux servers
- Detecting unauthorized file modifications from attacks or insider threats
- Meeting compliance requirements (PCI-DSS, HIPAA, SOC 2) for file integrity
- Implementing change detection for critical system files
- Establishing forensic baseline for incident response

## Prerequisites

### System Requirements
- Linux kernel 5.x or later
- Root or sudo access
- Minimum 100MB free disk space for AIDE database
- cron or systemd timer for scheduled checks

### Software Dependencies
- `aide` package (available in most distribution repositories)
- `curl` or `wget` for downloading configuration
- `grep`, `awk`, `sed` for log parsing
- `logger` for syslog integration

### Supported Distributions
- Ubuntu 20.04 LTS and later
- Debian 11 and later
- AlmaLinux 8/9
- RHEL 8/9
- Fedora 38 and later

## Steps

### 1. Install AIDE

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install -y aide

# RHEL/AlmaLinux/Fedora
sudo dnf install -y aide
```

### 2. Configure AIDE

The AIDE configuration file controls which directories and files are monitored, what attributes are checked, and how alerts are generated.

```bash
# Copy default configuration
sudo aideinit

# Edit configuration
sudo vi /etc/aide/aide.conf
```

Key configuration sections:
- `database`: Path to AIDE database file
- `database_out`: Path for updated database after checks
- `report`: Where to send reports
- `Rules`: Define what file attributes to check
- `Directories/Files`: Specify paths to monitor

### 3. Initialize Baseline Database

```bash
# Generate initial database
sudo aide --init

# Move database to production location
sudo mv /var/lib/aide/aide.db.new.gz /var/lib/aide/aide.db.gz
```

### 4. Run Manual Check

```bash
# Perform integrity check
sudo aide --check

# Check against a specific database
sudo aide --check --database=/var/lib/aide/aide.db.gz
```

### 5. Schedule Regular Checks

```bash
# Add to crontab for daily checks
sudo crontab -e

# Add this line for daily at 2 AM:
# 0 2 * * * /usr/bin/aide --check | /usr/bin/logger -t AIDE
```

### 6. Update Baseline After Authorized Changes

```bash
# Update database after system updates
sudo aide --update

# Install new database
sudo mv /var/lib/aide/aide.db.new.gz /var/lib/aide/aide.db.gz
```

## Verify

### Check AIDE Installation

```bash
# Verify AIDE version
aide --version

# Check configuration syntax
aide --config-check
```

### Test Baseline Creation

```bash
# Initialize database
sudo aide --init 2>&1 | head -20

# Verify database file exists
ls -la /var/lib/aide/aide.db.gz
```

### Test Integrity Check

```bash
# Run a check and capture output
sudo aide --check > /tmp/aide-check.log 2>&1

# Check exit code (0 = no changes, 1 = changes detected)
echo "Exit code: $?"

# View summary
grep -E "(Added|Removed|Changed)" /tmp/aide-check.log || echo "No changes detected"
```

### Verify Scheduled Job

```bash
# Check cron configuration
sudo crontab -l | grep aide

# View recent syslog entries
sudo journalctl -u cron -n 20 | grep AIDE || sudo grep AIDE /var/log/syslog | tail -10
```

## Rollback

### Uninstall AIDE

```bash
# Ubuntu/Debian
sudo apt-get remove aide

# RHEL/AlmaLinux/Fedora
sudo dnf remove aide
```

### Remove AIDE Database

```bash
# Remove database and logs
sudo rm -f /var/lib/aide/aide.db.gz
sudo rm -f /var/lib/aide/aide.db.new.gz
sudo rm -f /var/log/aide/*
```

### Disable Scheduled Checks

```bash
# Remove crontab entry
sudo crontab -e

# Remove the AIDE cron line
```

### Stop AIDE Service (if using systemd)

```bash
# If running as service (some distributions)
sudo systemctl stop aide
sudo systemctl disable aide
```

## Common errors

### "Error: Unable to open database"

**Problem:** AIDE cannot open the database file.

**Solution:**
```bash
# Check database file exists
ls -la /var/lib/aide/aide.db.gz

# Reinitialize database
sudo aide --init
sudo mv /var/lib/aide/aide.db.new.gz /var/lib/aide/aide.db.gz
```

### "Database not found"

**Problem:** AIDE database file is missing.

**Solution:**
```bash
# Initialize new database
sudo aide --init

# Verify file created
ls -la /var/lib/aide/
```

### "Configuration file syntax error"

**Problem:** Invalid syntax in aide.conf.

**Solution:**
```bash
# Validate configuration
sudo aide --config-check

# Restore default configuration
sudo cp /usr/share/aide/aide.conf.autogenerated /etc/aide/aide.conf
```

### "Permission denied" during check

**Problem:** Insufficient permissions to read monitored files.

**Solution:**
```bash
# Run as root
sudo aide --check

# Or adjust permissions on specific directories
sudo chmod +r /path/to/monitored/directory
```

### "Database out of sync" after updates

**Problem:** System updates changed files that AIDE flags as modified.

**Solution:**
```bash
# Update baseline after authorized changes
sudo aide --update

# Install new database
sudo mv /var/lib/aide/aide.db.new.gz /var/lib/aide/aide.db.gz
```

### "Out of memory" on large systems

**Problem:** AIDE consumes too much memory on systems with many files.

**Solution:**
```bash
# Limit monitored directories in aide.conf
# Exclude large directories
!/var/cache
!/var/log

# Run with reduced resource usage
nice -n 10 aide --check
```
